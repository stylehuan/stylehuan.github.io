Buffer对象

创建 Buffer对象的三种方式

方式一

参数

参数len为缓存区的长度

然后使用Buffer对象的fill方法，填充数据。该方法可以使用三个参数。

参数

  • value 是必须指定的参数，参数值为需要被写入的值。

  • offset 可选参数，用于指定从第一个字节开始写入被指定的值。默认是0.即从缓存区的起始位置写入

  • end 用于指定将值一直写到第几个字节处。默认是结尾。

方式二

直接使用数组来初始化缓存区

方式三

构造函数中直接使用一个字符串来初始化缓存区

参数

  • str 必须指定的参数。用于初始化缓存区的字符串。

  • encoding 可选参数，用于指定文字的编码格式。模式是“utf8”;

字符串与缓存区

字符串的长度与缓存区的长度完全不懂，一个取得是字符的个数，另一个取的是字节数。

buffer并不具备字符串的操作方法。只有一个slice方法。与string的slice相同。通过slice修改的取出的数据，也将会导致原缓存区的数据被修改。因为slice方法并不是copy缓存区的数据，而是共享内存区域。

Buffer对象与字符串对象之间的相互转换

  • toString方法

通过toString方法将Buffer对象中保存的数据转换为字符串。

第一个指定编码
第二、三个指定起止位置。

  • write方法。

向已创建的Buffer对象中写入字符串。也可以替换字符串

  • string: 必须参数，指定写入的字符串。

  • offset与lengtn用于指定字符串转换为字节数据后写入的位置。字节数据的写入位置为从第1+offset个字节开始到offset+length个字节为止。（例如。offset为3，length为6，写入的位置为从第4个字节开始到第9个字节位置，包括第四和第9字节。

  • encoding 指定字符串的编码格式。

Buffer对象与JSON对象之间的相互转换

在node中可以使用JSON.stringify方法将Buffer对象中保存的数据转换为一个字符串。也可以使用JSON.parse方法将一个经过转换后的字符串还原为一个数组。

复制缓存数据

如果需要复制一个Buffer对象的数据到另一个Buffer对象中，可以 使用Buffer对象的copy方法

  • targetBuffer 指定复制的目标Buffer对象

  • targetStart:指定从目标Buffer对象中第几个字节开始。

  • targetStart:从复制源的起始位置，默认是0.

  • targetEnd:直接复制源的结束位置

Buffer类的方法

在js中可以理解为静态方法。

  • isBuffer: 判断一个对象是否为一个BUffer对象
  • byteLength: 计算指定字符串的字节数。
  • isEncoding: 检测一个字符串是否为一个有效的编码格式的字符串。

CPU密集与I/O密集的区别

一张图说明CPU密集与I/O密集的区别

某个进程花费了一定的时间用在计算上的称之为计算密集型(cpu密集型)。而其他的在等待I/O上花费了大多是时间的称之为I/O密集型。

我的理解就是频繁的操作输入输出称之为I/O密集，此时，系统大部分的状况是CPU在等待I/O的读写，而CPU本身的loading不高。而CPU密集相反了。大部分时间是在运算。loading 基本在 100%吧。而I/O在很短的时间就可以完成。

node中的REPL交互式运行环境

node中为了方便开发者测试js代码，提供了一个名为REPL的可交互运行环境。在该运行环境中输入任何js表达式。当用户按下回车键后，REPL运行环境将显示该表达式的运行结果。

在命令行窗口中，输入”node”按下回车，即可进入REPL运行环境。

变量操作

在REPL环境中我们可以输入任何js表达式，执行并输出结果.只是变量声明不需要var.在REPL环境内部使用了eval函数来执行表达式的结果。

在REPL环境中，输入变量名直接打印结果，输入对象名，直接显示该对象的属性名及值。如果属性值是一个函数，那么将显示 [function];

下划线使用

在REPL中可以通过下划线(“_”)来访问最近使用的表达式。

运行函数

当一个表达式未书写完成是，REPL环境将为该表达式的每一行添加英文省略符。知道表达式书写完成。

上下文

在node模块中可以使用start方法来开启REPL的运行环境。该方法包含一个参数，用来指定开启运行环境的选项。具体的选项暂不讨论。start方法返回了REPL的运行环境。可以为这个环境指定一个上下文对象。然后将变量保存到该对象上，那么该变量就是该上下文中的全局变量。

基础命令

在node中提供了一些基础的命令便于我们更好的使用REPL环境.这些命令都是以.点开始的。

  • .break:

跳出当前正在输入的表达式并回到一开始状态。（也可使用ctrl +c）

  • .clear: 清除上下文中所有变量和函数

  • .exit: 退出REPL环境。

  • .help: 显示REPL环境中的所有基础命令

  • save: 把当前环境中输入的所有表达式保存到一个文件中。

  • .load:把已保存的文件，加载到环境中。
